---
title: "Get started with ChargeGo"
description: "Go from zero to your first charging session in five steps: register, add a vehicle, find a station, reserve a charger, and start charging."
---

This guide walks you through the core ChargeGo workflow using the REST API. By the end you will have a registered account, a vehicle on file, a confirmed reservation, and an active charging session. All requests authenticate with HTTP Basic Auth — replace `yourUsername` and `yourPassword` with your credentials in every example below.

<Note>
  Prefer clicking over curl? Every step below can also be completed through the web UI at [https://charge-go.duckdns.org/](https://charge-go.duckdns.org/).
</Note>

<Steps>
  <Step title="Register your account">
    Create an account by submitting your desired username and password to the registration endpoint. Usernames must be 3–50 characters and passwords must be 6–100 characters.

    <CodeGroup>
      ```bash curl
      curl -X POST https://charge-go.duckdns.org/ui/register \
        -H "Content-Type: application/x-www-form-urlencoded" \
        -d "username=jane&password=secret123&confirmPassword=secret123"
      ```
    </CodeGroup>

    A successful registration redirects you to the login page. You can then authenticate all subsequent API requests using your new credentials as HTTP Basic Auth.

    <Tip>
      You can also register through the browser at [https://charge-go.duckdns.org/ui/register](https://charge-go.duckdns.org/ui/register).
    </Tip>
  </Step>

  <Step title="Add your vehicle">
    Register the EV you want to charge. ChargeGo uses the connector type you provide here to validate compatibility when you make a reservation — so make sure it matches your vehicle's actual connector.

    **Supported connector types:** `CCS`, `TYPE2`, `CHADEMO`

    <CodeGroup>
      ```bash curl
      curl -X POST https://charge-go.duckdns.org/vehicles \
        -u yourUsername:yourPassword \
        -H "Content-Type: application/json" \
        -d '{
          "brand": "Tesla",
          "model": "Model 3",
          "batteryCapacity": 75.0,
          "connectorType": "CCS",
          "plateNumber": "34ABC123"
        }'
      ```
    </CodeGroup>

    The response includes an `id` — keep it handy, you will need it when creating a reservation.

    ```json
    {
      "id": 1,
      "brand": "Tesla",
      "model": "Model 3",
      "batteryCapacity": 75.0,
      "connectorType": "CCS",
      "plateNumber": "34ABC123"
    }
    ```
  </Step>

  <Step title="Discover available stations">
    Search for charging stations that match your vehicle's connector type. All query parameters are optional — omit any you don't need.

    | Parameter | Description |
    |---|---|
    | `connectorType` | Filter by `CCS`, `TYPE2`, or `CHADEMO` |
    | `powerOutput` | Filter by power tier |
    | `maxPricePerKwh` | Upper price limit in your currency |
    | `status` | `AVAILABLE`, `OCCUPIED`, or `OFFLINE` |
    | `userLatitude` / `userLongitude` | Your location for distance calculation |

    <CodeGroup>
      ```bash curl
      curl -X GET "https://charge-go.duckdns.org/stations/discovery?connectorType=CCS&status=AVAILABLE&userLatitude=41.015&userLongitude=28.979" \
        -u yourUsername:yourPassword
      ```
    </CodeGroup>

    Each result includes station details, available charger count, minimum price per kWh, and — if you passed your coordinates — the distance in kilometres.

    ```json
    [
      {
        "id": 3,
        "name": "Kadıköy Charging Hub",
        "address": "Moda Cad. No:12, Kadıköy",
        "latitude": 40.9903,
        "longitude": 29.0224,
        "status": "AVAILABLE",
        "totalChargerCount": 4,
        "availableChargerCount": 2,
        "minPricePerKwh": 3.5,
        "connectorTypes": ["CCS"],
        "distanceKm": 3.12,
        "chargers": [
          {
            "id": 10,
            "chargerCode": "KDK-01",
            "chargerType": "DC",
            "connectorType": "CCS",
            "powerOutput": "KW_50",
            "pricePerKwh": 3.5,
            "status": "AVAILABLE"
          }
        ]
      }
    ]
    ```

    Note the `id` of the charger you want to book (e.g. `10` above).
  </Step>

  <Step title="Create a reservation">
    Reserve a specific charger for a time window. Provide the vehicle ID from step 2 and the charger ID from step 3. Times are ISO 8601 local datetime values.

    ChargeGo will reject the request if:
    - The charger's connector type does not match your vehicle
    - The time window overlaps an existing reservation on that charger
    - The charger is not available

    <CodeGroup>
      ```bash curl
      curl -X POST https://charge-go.duckdns.org/reservations \
        -u yourUsername:yourPassword \
        -H "Content-Type: application/json" \
        -d '{
          "vehicleId": 1,
          "chargerId": 10,
          "startTime": "2026-05-10T14:00:00",
          "endTime": "2026-05-10T15:30:00"
        }'
      ```
    </CodeGroup>

    A successful response returns the reservation with status `PENDING`:

    ```json
    {
      "id": 42,
      "vehicleId": 1,
      "chargerId": 10,
      "startTime": "2026-05-10T14:00:00",
      "endTime": "2026-05-10T15:30:00",
      "status": "PENDING"
    }
    ```

    Save the reservation `id` — you will need it to start your session.
  </Step>

  <Step title="Start your charging session">
    When you arrive at the charger and plug in, start the session against your reservation. Pass the reservation ID and, optionally, your current battery level as a percentage (0–100).

    <CodeGroup>
      ```bash curl
      curl -X POST "https://charge-go.duckdns.org/sessions/start?reservationId=42&startBatteryLevel=20" \
        -u yourUsername:yourPassword
      ```
    </CodeGroup>

    The response confirms the session is active and shows your starting energy in kWh:

    ```json
    {
      "id": 7,
      "reservationId": 42,
      "chargerId": 10,
      "startBatteryLevel": 20.0,
      "startEnergyKwh": 15.0,
      "unitPricePerKwh": 3.5,
      "startedAt": "2026-05-10T14:02:33",
      "status": "ACTIVE"
    }
    ```

    When you finish charging, end the session with your final battery level:

    <CodeGroup>
      ```bash curl
      curl -X POST "https://charge-go.duckdns.org/sessions/7/end?endBatteryLevel=85" \
        -u yourUsername:yourPassword
      ```
    </CodeGroup>

    The final response includes your total kWh consumed and the cost for the session:

    ```json
    {
      "id": 7,
      "reservationId": 42,
      "chargerId": 10,
      "startBatteryLevel": 20.0,
      "endBatteryLevel": 85.0,
      "chargedPercentage": 65.0,
      "startEnergyKwh": 15.0,
      "endEnergyKwh": 63.75,
      "consumedKwh": 48.75,
      "unitPricePerKwh": 3.5,
      "totalCost": 170.63,
      "startedAt": "2026-05-10T14:02:33",
      "endedAt": "2026-05-10T15:28:10",
      "status": "COMPLETED"
    }
    ```
  </Step>
</Steps>

## What's next

- **Manage your vehicles** — update or remove vehicles with `PUT /vehicles/{id}` and `DELETE /vehicles/{id}`.
- **Cancel a reservation** — use `PATCH /reservations/{id}/cancel` if your plans change before the session starts.
- **Review past sessions** — `GET /sessions` returns all your charging sessions with full cost breakdowns.
